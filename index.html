Problem Statement 104 — Max Deck Size

✦
Parameters
Spectator picks M cards; friend hides 1, arranges M−1
Max adjacent swaps the spectator may execute
Maximum Deck Size — N
—
How the Algorithm Works Your friend encodes the hidden card as one of (M−1)! permutations of the visible cards. The spectator corrupts it with ≤ S inversions. The number of permutations reachable within S swaps is T(n, S) — computed via DP on inversion tables. The answer is ⌊(M−1)! / T(n,S)⌋, the maximum deck size where every possible hidden card maps to a unique, distinguishable signal.
